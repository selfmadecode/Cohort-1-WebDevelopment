The <img> Tag

This note explains how to show images on web pages using the <img> element. No prior knowledge required. Copy examples and experiment in a text editor + browser.

What is <img>?

The <img> element inserts an image into an HTML page. It's an inline replaced element (the browser replaces it with an image graphic). Example:

<img src="logo.png" alt="Company logo">

Important rules

  • Every image should have an alt attribute for accessibility and SEO.
  • <img> is self-closing in HTML: you don't need a closing tag.
  • If the src file is missing or wrong, the browser shows a broken-image icon or nothing; alt text is used by screen readers.

All the main <img> attributes (explained)

AttributeWhat it doesExample
srcURL or path to the image file (required for visible images).<img src="images/photo.jpg" alt="..."/>
altAlternative text used by screen readers and shown when image fails to load — required for accessibility.alt="A red bicycle parked by a tree"
width / heightIntrinsic width/height in pixels (numbers). Helps layout stability. CSS can also control size.width="300" height="200"
srcsetMultiple source candidates for responsive images (used with sizes).srcset="small.jpg 480w, medium.jpg 800w, large.jpg 1600w"
sizesHints to the browser how wide the image will appear at different viewport widths.sizes="(max-width:600px) 100vw, 50vw"
loadingLazy-load control: lazy defers loading until near viewport, eager loads immediately.loading="lazy"
decodingImage decoding hint: async (default), sync (block), or auto.decoding="async"
crossoriginUsed when the image is fetched across origins and you need CORS behavior: anonymous or use-credentials.crossorigin="anonymous"
referrerpolicySpecifies the referrer header policy for the image request (e.g. no-referrer).referrerpolicy="no-referrer"
usemapAssociate the image with an <map> for clickable regions (image map).usemap="#mymap"
ismapCreates a server-side image map (rarely used). Clicking sends coordinates to server.<img src="map.png" ismap>
titleTooltip text shown on hover (not a substitute for alt).title="More info"
longdescOld/obsolete method for long descriptions (deprecated; use linked pages or ARIA instead).(not recommended)
role / ARIAFor accessibility roles — used carefully when semantically necessary.role="img" aria-label="... (if no alt)"
draggableWhether image can be dragged: true or false.draggable="false"

Notes on a few attributes

  • alt is critical. If image conveys meaning, write succinct descriptive alt text. If purely decorative, use alt="" (empty) so screen readers ignore it.
  • width and height values help the browser reserve layout space and avoid layout shifts when the image loads (improves Core Web Vitals).
  • srcset + sizes let you serve different images (small/large) for different screen sizes — essential for responsive images and performance.
  • loading="lazy" is a simple way to lazy-load off-screen images to speed up initial page load.
  • crossorigin and referrerpolicy affect privacy and whether you can access pixel data (canvas) from images loaded from other origins.

Paths: Locating images locally vs remote

Images are fetched from a URL. There are several kinds of src values:

  1. Relative path (relative to the HTML file):
    2. <!-- same folder as the HTML file -->
<img src="image.png" alt="Same folder image">

<!-- subfolder called images -->
<img src="images/photo.jpg" alt="Photo in images folder">

<!-- parent folder (go up one level) -->
<img src="../other-folder/pic.jpg" alt="From parent folder">
  2. Root-relative path (starts at website root): 
    <img src="/assets/logo.png" alt="Site logo (root-relative)">
  3. Absolute URL (remote/CDN): 
    <img src="https://cdn.example.com/images/photo.jpg" alt="From a CDN">
  4. File URL (local files, only useful in local testing): 
    <!-- Not recommended for deployed sites -->
<img src="file:///C:/Users/me/Pictures/photo.jpg" alt="Local file">

Using the command line to find images (basic cd examples)

If you edit files in a terminal, you use cd to move between folders. Here are quick examples for a beginner (these commands are run in a terminal / PowerShell / macOS Terminal):

# List files in current directory (Linux/macOS)
ls

# Windows Command Prompt
dir

# Change directory into a folder named "project"
cd project

# Go up one folder
cd ..

# Example: you are in /home/me/project/pages and image is in /home/me/project/images
# In HTML file inside pages, you reference the image with ../images/pic.jpg
# Terminal navigation:
cd /home/me/project/pages
# to go to images folder:
cd ../images
Tip: When your HTML file and images are in the same project, think about folder structure first: 
project/
  index.html
  images/
    hero.jpg
    logo.png
  css/
    style.css
Then from index.html you use src="images/hero.jpg".

Examples: local, CDN, relative, and root paths

<!-- local (relative) example -->
<img src="./same-path-image.png" alt="Example image in same folder" width="400">

<!-- parent folder example -->
<img src="../images/from-parent.jpg" alt="From parent folder">

<!-- root-relative -->
<img src="/static/logo.svg" alt="Root relative logo">

<!-- CDN (remote) example -->
<img src="https://cdn.example.com/assets/picture.webp" alt="From CDN" width="300">

Responsive images: srcset, sizes and <picture>

Use these when you want different images for different devices (mobile, tablet, desktop) or different image formats (webp vs jpeg).

<!-- srcset + sizes -->
<img
  src="images/photo-medium.jpg"
  srcset="images/photo-small.jpg 400w,
          images/photo-medium.jpg 800w,
          images/photo-large.jpg 1600w"
  sizes="(max-width:600px) 100vw, (max-width:1200px) 50vw, 800px"
  alt="Scenic view">

<!-- picture for format switching -->
<picture>
  <source srcset="images/photo.webp" type="image/webp">
  <img src="images/photo.jpg" alt="Scenic view (fallback to jpeg)">
</picture>

Accessibility checklist

  • Always include alt text. If decorative, use alt="".
  • Use meaningful alt for content images — describe what the image shows, not file name.
  • Provide captions or long descriptions on the page if the image conveys complex information.

Performance & best practices

  • Use appropriately sized images; avoid sending huge images to mobile devices.
  • Compress images (WebP/AVIF are modern formats with better compression).
  • Use width and height attributes (or CSS aspect-ratio) to avoid layout shift.
  • Use loading="lazy" for below-the-fold images to speed up initial load.
  • Use a CDN for public sites to speed global delivery and reduce latency.

Error handling

If an image fails to load, the browser shows broken icon and the alt text becomes important. You can also use JavaScript to handle onerror and provide a fallback:

<img src="maybe-missing.jpg" alt="Fallback example" onerror="this.src='fallback.jpg';">

Full page example (put this in a file and open it)

Save as img-lecture-demo.html alongside an images folder (or change paths to match your files):

<!doctype html>
<html lang="en">
<head>
  <meta charset="utf-8">
  <meta name="viewport" content="width=device-width,initial-scale=1">
  <title>Img tag demo</title>
</head>
<body>
  <h1>IMG tag demo</h1>

  <!-- Local (relative) -->
  <img src="images/example.jpg" alt="Example photo of a mountain" width="600" height="400">

  <!-- CDN -->
  <img src="https://via.placeholder.com/300" alt="Placeholder from remote CDN">

  <!-- Responsive srcset -->
  <img
    src="images/photo-medium.jpg"
    srcset="images/photo-small.jpg 480w, images/photo-medium.jpg 800w, images/photo-large.jpg 1600w"
    sizes="(max-width:600px) 100vw, 800px"
    alt="Responsive example">

  <!-- Lazy load -->
  <img src="images/large.jpg" alt="Lazy loaded" loading="lazy">
</body>
</html>

Quick summary

  • <img src=... alt=...> — the basics.
  • Use proper paths: relative (./, ../), root (/), or absolute/remote (https://...)
  • alt is essential for accessibility. Use loading="lazy" for off-screen images.
  • Learn srcset + sizes or <picture> for responsive images and format switching.
